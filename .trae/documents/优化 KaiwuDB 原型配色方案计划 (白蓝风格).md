# 优化 KaiwuDB 原型配色方案计划 (白蓝风格)

我将基于您的要求，将现有原型的配色方案从深色科技风重构为**清爽白蓝风格**。

## 1. 配色系统重构
- **主色调**:
  - **背景色**: 使用纯白 (`#FFFFFF`) 作为主背景，浅灰 (`#F5F7FA`) 作为次级背景（用于区分区块）。
  - **品牌蓝**: 使用 KaiwuDB 风格的科技蓝 (`#0056D2` 或类似) 作为主色，用于按钮、高亮文本和图标。
  - **辅助蓝**: 使用浅蓝 (`#E6F0FF`) 作为悬停态或装饰性背景。
  - **文字色**: 深灰 (`#1D1D1F`) 用于标题，中灰 (`#6E6E73`) 用于正文，确保高对比度。

## 2. 界面元素更新策略
- **导航栏**:
  - 从深色玻璃拟态改为**白色玻璃拟态** (`rgba(255, 255, 255, 0.9)`)，底部增加细微阴影。
  - 链接颜色改为深灰，悬停变蓝。
- **Hero 区域**:
  - 背景改为**浅蓝渐变** (`linear-gradient(135deg, #F5F7FA 0%, #E6F0FF 100%)`)。
  - 增加蓝色科技粒子或波浪 SVG 作为装饰，保持科技感但不沉重。
  - 文字颜色反转为深色。
- **卡片组件**:
  - 背景改为**纯白**，增加轻微的阴影 (`box-shadow: 0 4px 20px rgba(0, 0, 0, 0.05)`)。
  - 悬停时提升阴影深度并出现蓝色边框。
- **Tab 切换区**:
  - 选中状态使用品牌蓝底白字，未选中状态使用灰底深字。
- **底部 Footer**:
  - 改为深蓝色背景 (`#0A192F` 保持深色以稳住底部) 或 深灰背景，文字为白色，形成视觉收尾。

## 3. 执行步骤
1.  **更新 CSS 变量**: 修改 `style.css` 中的 `:root` 定义，替换颜色系统。
2.  **调整组件样式**: 针对 Hero、Navbar、Card、Tabs 等组件适配新配色。
3.  **无障碍检查**: 确保所有文本与背景的对比度符合 WCAG AA 标准 (4.5:1)。
4.  **文档输出**: 创建 `DESIGN_GUIDE.md` 记录配色规范。

## 4. 交付物
- 更新后的 `css/style.css`
- 配色规范文档 `DESIGN_GUIDE.md`
- 保持运行的预览服务
